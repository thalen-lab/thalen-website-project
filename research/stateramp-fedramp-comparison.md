## Executive Summary

**FedRAMP** (Federal Risk and Authorization Management Program) is a mandatory program for cloud service providers (CSPs) working with federal agencies, ensuring stringent security and privacy protections.

**StateRAMP** was launched as a counterpart to FedRAMP, providing similar security assurances for cloud services used by state and local governments. It leverages FedRAMP standards to create a harmonized approach tailored to state and local government needs.

---

## Understanding FedRAMP

### Purpose
FedRAMP compliance is a mandate for all cloud service providers that wish to work with federal agencies. It ensures that the data these providers handle meets stringent security and privacy protections.

### FedRAMP Process Overview

The FedRAMP process involves three main steps:

#### 1. Initiating a Security Assessment
- Select a Third Party Assessment Organization (3PAO) certified by FedRAMP
- 3PAO conducts comprehensive assessment of CSP's systems
- Ensures systems meet required security controls based on NIST guidelines
- Security assessment report outlines the security posture of CSP's services

#### 2. Obtaining Provisional Authorization from Joint Authorization Board (JAB)
- JAB consists of representatives from:
  - Department of Homeland Security (DHS)
  - General Services Administration (GSA)
  - Department of Defense (DoD)
- Provisional Authorization (PA) means cloud service is deemed secure for use across federal agencies
- Subject to specific conditions
- CSPs typically pursue this by offering services with broad applicability across multiple agencies

#### 3. Receiving an Authorization to Operate (ATO) from an Agency
- ATO issued directly by federal agency intending to use the cloud service
- Agency reviews 3PAO assessment
- Based on agency's risk posture, decides whether to authorize cloud service
- Can be faster path for CSPs with specific agency ready to sponsor their service

### Target Audience
- Federal government agencies
- Cloud service providers serving federal government

### Characteristics
- More extensive and resource-intensive process
- Mandatory requirement for federal agencies
- Provides single set of mandatory requirements for all federal agencies
- Generally more expensive due to strict requirements and review process
- More stringent requirements than StateRAMP

## Understanding StateRAMP

### Purpose
StateRAMP provides similar security assurances for cloud services used by state and local governments. It aims to simplify security authorization by providing a single authorization that can be used across multiple states.

### StateRAMP Process Overview
- Process mirrors FedRAMP's but adapted to smaller governmental bodies' scale and specific needs
- Providers already FedRAMP authorized can often use existing security packages to achieve StateRAMP compliance
- Reduces duplication of effort
- Leverages FedRAMP standards for harmonized approach

### Target Audience
- State and local governments
- State agencies
- Educational institutions
- Cloud service providers serving state/local government

### Characteristics
- Governed by non-profit organization (not affiliated with U.S. federal government)
- Less expensive and less resource-intensive compared to FedRAMP
- Requirements can vary from state to state, but goal is unified program
- "Fast Track" option for those already FedRAMP authorized
- Authorize once, use by multiple states

### Benefits for Businesses
- Increases marketability to state and local government clients
- Demonstrates compliance with respected security standards
- No need for separate assessments (if already FedRAMP authorized)
- Builds trust with government clients
- Provides competitive edge in securing contracts
- Particularly important for businesses operating primarily at state or local level

## State-Specific Adaptations of StateRAMP

States are developing their own versions of StateRAMP, adapting the framework to meet local government requirements:

### AZRAMP (Arizona RAMP)
- Designed specifically for Arizona
- Aligns with Arizona's unique legislative and regulatory environment
- Emphasizes compliance with local data protection laws
- Aims to bolster state's defenses against emerging cyber threats

### GA-RAMP (Georgia RAMP)
- Extends StateRAMP principles to meet Georgia's state agencies' needs
- Focuses on integrating state-level security policies with nationally recognized standards
- Ensures cloud services are secure and compliant with Georgia's specific requirements

### OH-RAMP (Ohio RAMP)
- Tailored to Ohio's specific regulatory framework
- Ensures cloud-based services adhere to security protocols protecting against state-specific cyber vulnerabilities
- Emphasizes collaboration between public and private sectors
- Enhances state-wide cybersecurity resilience

### TX-RAMP (Texas RAMP)
- Provides robust framework for Texas
- Requires compliance with comprehensive set of security controls
- Designed to address cybersecurity challenges faced by Texas state agencies
- Includes heightened measures for data privacy and security
- Caters to Texas's extensive public sector

## Key Differences: StateRAMP vs. FedRAMP

| Feature | StateRAMP | FedRAMP |
|---------|-----------|---------|
| **Primary Customer** | State and local governments, including state agencies and educational institutions | Federal government agencies |
| **Governing Body** | Non-profit organization not affiliated with U.S. federal government | Program mandated by federal government, managed by GSA |
| **Authorization Process** | Aims to simplify security authorization; single authorization used across multiple states | More extensive and resource-intensive process; mandatory requirement for federal agencies |
| **Implementation** | Requirements can vary from state to state, but goal is unified program | Provides single set of mandatory requirements for all federal agencies |
| **Cost** | Generally less expensive and less resource-intensive | Generally more expensive and more resource-intensive due to strict requirements |
| **Scope** | State and local government use | Federal government use |
| **Flexibility** | "Fast Track" option for FedRAMP authorized providers | Standard process for all |

## Similarities Between StateRAMP and FedRAMP

Both programs share several key characteristics:

1. **Standardized Cybersecurity Approach**: Both designed to standardize and streamline cybersecurity for cloud services
2. **NIST 800-53 Controls**: Both use NIST 800-53 controls as security baseline
3. **Third-Party Assessment**: Both require Third Party Assessment Organizations (3PAOs) for security assessments
4. **Continuous Monitoring**: Both require continuous monitoring of security posture
5. **Cloud Service Focus**: Both specifically designed for cloud service providers
6. **Security Standards**: Both provide rigorous security standards for government use

## Strategic Considerations

### For Cloud Service Providers

**Choose FedRAMP when:**
- Targeting federal government agencies as primary customers
- Offering services with broad applicability across multiple federal agencies
- Have resources for more extensive, expensive authorization process
- Need to meet mandatory federal requirements

**Choose StateRAMP when:**
- Targeting state and local governments as primary customers
- Already have FedRAMP authorization (Fast Track available)
- Have limited resources for authorization process
- Want to serve multiple states with single authorization

**Consider Both when:**
- Serving both federal and state/local government markets
- FedRAMP authorization can streamline StateRAMP process
- Maximizing government market opportunities

### For Government Agencies

**FedRAMP is required for:**
- All federal government agencies
- Handling sensitive federal data
- Federal contracts and procurement

**StateRAMP is appropriate for:**
- State and local government agencies
- Educational institutions
- Regional government collaborations
- State-specific compliance requirements
